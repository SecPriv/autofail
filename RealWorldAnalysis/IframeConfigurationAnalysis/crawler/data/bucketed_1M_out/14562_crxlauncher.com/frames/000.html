Frequently Asked Questions

What is CRXLauncher and what does it do?

CRXLauncher enables users to seamlessly install Chrome extensions from Opera Addons and CRX files into Chromium-based browsers such as Chrome, Edge, and Opera. It supports a wide range of Chrome APIs to ensure high compatibility across extensions.

How can I install an extension from the Opera Addons Webstore?

To install an Opera extension using CRXLauncher:

- First, install CRXLauncher from the Chrome Web Store.

- Navigate to the Opera Addons Webstore.

- Click the Install with CRXLauncher button on the addon's sidebar.

- The installation will begin automatically via the CRXLauncher interface.

- Once confirmed, the extension will be available in your browser.

Can I install a CRX file using CRXLauncher?

Yes, you can install local CRX files using these steps:

- Install CRXLauncher from the Chrome Web Store.

- Open the CRXLauncher settings page.

- Click Choose File and upload the desired .crx file.

- If compatible, the extension will be installed.

Note: Automatic updates are not available for CRX file installations.

Does CRXLauncher support automatic updates?

CRXLauncher provides automatic updates only for extensions installed from the Opera Addons Webstore or hosted sources. CRX files do not support auto-update functionality.

Which manifest versions does CRXLauncher support?

CRXLauncher is fully compatible with both Manifest v2 and Manifest v3, allowing developers flexibility in extension design.

What manifest features are supported by CRXLauncher?

CRXLauncher supports the following manifest elements:

- default_locale

- icons

- action / browser_action

- options_ui

- background (including service workers and background pages)

- content_scripts

- web_accessible_resources

How does CRXLauncher handle content scripts?

Content scripts are executed at document_end and are not sandboxed. The chrome.locale API and chrome.runtime.getURL() (returning base64-encoded URLs) are fully supported.

Are web accessible resources supported?

Yes, partially. These resources are available only via chrome.runtime.getURL() and returned as base64-encoded data URLs.

What optional permissions does CRXLauncher support?

The following permissions are supported and requested during installation:

  • http:///
  • https:///
  • alarms
  • storage
  • tabs

Which Chrome APIs are compatible with CRXLauncher?

CRXLauncher supports these APIs to maximize extension compatibility:

  • chrome.alarms
  • chrome.action
  • chrome.browserAction
  • chrome.downloads
  • chrome.i18n
  • chrome.permissions
  • chrome.runtime
  • chrome.storage
  • chrome.tabs
  • chrome.windows

Does CRXLauncher support XMLHttpRequest?

Yes. Although Chromium's sandbox restricts direct usage, CRXLauncher provides a custom polyfill that translates XMLHttpRequest calls through its API without requiring code changes.

Can my extension use localStorage?

Direct use of localStorage is restricted. However, CRXLauncher includes a polyfill that maps localStorage functionality to the Chrome storage API, ensuring your extension continues to operate as expected.
